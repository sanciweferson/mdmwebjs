console.log(name, typeof(name)) → "Sanciweferson" tipo string

Em JavaScript, todo valor possui um tipo. O tipo define como o JavaScript entende esse valor e quais operações podem ser feitas com ele.

O tipo string é usado para representar textos. Qualquer valor escrito entre aspas (" ", ' ' ou ` `) é tratado como string.

A variável name armazena o texto "Sanciweferson", portanto o JavaScript identifica automaticamente esse valor como uma string.

O operador typeof é usado para verificar o tipo de um valor em tempo de execução. Neste caso, ele retorna "string".

O tipo number é usado para representar valores numéricos em JavaScript. Diferente de outras linguagens, não existe separação entre números inteiros e decimais: todos são do tipo number.

Isso significa que valores como 10, 3.14 e -5 pertencem ao mesmo tipo. Internamente, o JavaScript usa um padrão chamado IEEE 754 para representar números.

Números do tipo number permitem operações matemáticas como soma, subtração, multiplicação e divisão. Por exemplo: 10 + 5 resulta em 15.

O operador typeof, quando aplicado a um número, retorna "number", confirmando que o valor armazenado é tratado como um dado numérico pelo JavaScript.

É importante lembrar que números escritos entre aspas, como "10", não são number, mas sim string, mesmo parecendo números.

4. Data Types — Null

console.log(aptNumber, typeof(aptNumber)) → "null" tipo objecto

O valor null representa a ausência intencional de um valor. Ele é usado quando o programador quer dizer explicitamente: “aqui não existe nada, de propósito”.

Diferente de undefined, que indica que uma variável ainda não recebeu valor, null é atribuído manualmente. Ele não acontece por acidente.

Em termos de tipo, o JavaScript retorna "object" ao usar typeof null. Isso é um erro histórico da linguagem, mantido por compatibilidade, e não significa que null seja realmente um objeto.

O uso de null é comum para indicar estados como: valor não definido ainda, resposta vazia de uma busca, ou quando um dado foi propositalmente removido.

Conceitualmente, null significa: a variável existe, foi criada, mas aponta para “nenhum valor”.

3. Data Types — Boolean

console.log(haskids, typeof(haskids)) → "true" tipo boolean

O tipo boolean representa um valor lógico, que pode ser apenas true ou false. Ele é a base de toda tomada de decisão em JavaScript.

Valores booleanos são usados principalmente em estruturas de controle como if, else, while e expressões condicionais.

O operador typeof, quando aplicado a um valor boolean, retorna "boolean", indicando que o JavaScript está lidando com um valor lógico.

Em JavaScript, expressões de comparação como 10 > 5 ou 5 === 3 sempre resultam em um valor boolean.

É importante entender que o JavaScript também converte outros valores para boolean em certos contextos. Valores como 0, "", null, undefined e NaN são tratados como false, enquanto a maioria dos outros valores é tratada como true.
