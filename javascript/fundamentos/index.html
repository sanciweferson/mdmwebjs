Guia de Fundamentos JS

1.1 O que é JavaScript

  • Linguagem de Programação

    O "cérebro" da web. Define o comportamento e como a página reage a ações.

  • Alto Nível

    Sintaxe próxima da linguagem humana, facilitando a escrita sem gerenciar memória manualmente.

  • Interpretada / JIT

    O código é analisado, compilado e otimizado em tempo de execução pelo motor JavaScript (como o V8 do Chrome).

  • Onde roda?

    Em qualquer lugar: Navegadores, Servidores (Node.js), Mobile e até IoT.

1.2 Linha do Tempo

  • 1995

    Nascimento: Criado por Brendan Eich em apenas 10 dias na Netscape.

  • 1997

    Padronização: Torna-se ECMAScript.

  • 2009

    Node.js: JavaScript chega ao backend.

  • 2015

    ES6: Grande evolução da linguagem.

1.3 Por que o nome JavaScript

  • Marketing

    O nome aproveitou a popularidade do Java nos anos 90.

  • JavaScript ≠ Java

    São linguagens completamente diferentes.

  • Nome oficial

    ECMAScript é a especificação da linguagem.

1.4 O que JavaScript não é

  • Não é Java

    Não compartilha arquitetura nem execução.

  • Não é só frontend

    Também roda em servidores, desktop e IoT.

  • Não é simples

    Possui modelo assíncrono e engines avançadas.

2️⃣ JavaScript e o Ambiente

2.1 JavaScript Engine

  • O que é

    Software que interpreta, compila e executa JavaScript.

  • Exemplos

    V8, SpiderMonkey, JavaScriptCore.

  • Função

    Transformar código em instruções executáveis (JIT).

2.2 JavaScript Runtime

  • Engine x Runtime

    Engine executa. Runtime fornece APIs.

  • Browser

    DOM, eventos, Web APIs.

  • Node.js

    Arquivos, rede, processos.

2.3 O papel do navegador

  • DOM

    Representação do HTML.

  • BOM

    window, history, location.

  • Web APIs

    fetch, setTimeout, eventos.

3️⃣ Arquivo .js e o Processo de Execução

3.1 O que é um arquivo .js

  • Source file

    Um arquivo .js é um arquivo de origem que contém código JavaScript.

  • Source code

    O conteúdo escrito pelo desenvolvedor, antes de qualquer interpretação ou execução pela engine.

  • Código como texto

    Inicialmente, o JavaScript existe apenas como texto puro, sem significado computacional até ser processado.

3.2 Quando o JavaScript começa a existir

  • Script loading

    O JavaScript passa a existir quando o arquivo é carregado pelo navegador ou Node.js.

  • Inclusão no HTML

    Via tag <script>, embutido ou externo.

  • Execução no Node.js

    O JavaScript passa a existir quando carregado pelo runtime via linha de comando.

3.3 Parsing e leitura do código

  • Análise léxica

    O código é lido caractere por caractere para identificar estruturas válidas.

  • Tokens

    O código é quebrado em unidades mínimas significativas: palavras-chave, operadores e identificadores.

  • Parsing

    Os tokens são organizados conforme as regras gramaticais da linguagem.

  • AST (Abstract Syntax Tree)

    Estrutura em árvore que representa o código de forma abstrata para execução e otimização.

4️⃣ Execution Context (O Coração do JavaScript)

4.1 O que é Execution Context

  • Definição formal

    Ambiente interno criado pela engine para armazenar, organizar e executar o código.

  • Por que ele existe

    Define onde variáveis, funções e o valor de this vivem durante a execução.

  • Por que nada roda fora dele

    Todo código JS precisa de um Execution Context para ser interpretado e executado.

4.2 Tipos de Execution Context

  • Global Execution Context

    Contexto criado quando o programa inicia. Existe apenas um por aplicação.

  • Function Execution Context

    Criado cada vez que uma função é invocada, com escopo e variáveis locais.

  • Eval Execution Context

    Criado pela função eval(), raramente usado e desaconselhado.

4.3 Fases do Execution Context

  • Creation Phase

    Fase em que a engine cria o contexto, registra variáveis e define this.

  • Execution Phase

    Fase em que o código é executado linha por linha.

5️⃣ Léxico, Escopo e Ambiente

5.1 O Conceito de "Léxico"

Léxico refere-se ao escopo definido pela posição do código. Ou seja, onde você escreve suas declarações determina seu alcance.

  • Escopo Estático

    O JS decide o alcance de variáveis ao escrever o código, não ao executar.

  • Escopo Léxico

    Funções "lembram" do ambiente onde foram criadas e mantêm acesso às variáveis daquele escopo.

  • Escopo Léxico vs Dinâmico

    JS usa escopo léxico. Escopo dinâmico, baseado na pilha de chamadas, não é adotado.

5.2 Ambiente Léxico (Lexical Environment)

O ambiente léxico é a estrutura interna do JS responsável por registrar identificadores, o escopo onde foram criados e a referência ao valor armazenado na memória.

  • Environment Record

    Armazena todas as declarações de variáveis, funções e parâmetros dentro de um escopo.

  • Outer Lexical Environment

    Referência ao ambiente externo, permitindo acesso a variáveis de escopos superiores.

  • Scope Chain

    Cadeia de ambientes léxicos usada para resolver identificadores não encontrados no escopo atual.

5.3 Variable Environment

  • Diferença para Lexical Environment

    Usado especificamente para var, enquanto o Lexical Environment gerencia let, const e funções em blocos.

  • Funciona como "lado B" do ambiente léxico: explica porque var não respeita blocos if ou for, apenas funções.
