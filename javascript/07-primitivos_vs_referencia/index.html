Primitivos vs Referência em JavaScript

Valores Primitivos

number, string, boolean, null, undefined, symbol e bigint. São imutáveis: não mudam "por dentro". Para alterar, é necessário substituir o valor inteiro (reatribuição). 


      

    


    

      
Valores de Referência

      

        Objetos, arrays e funções não guardam o valor direto.
        Eles guardam um endereço de memória. Várias variáveis podem
        apontar para o mesmo objeto — e aí uma muda e a outra sente junto.
      

      

      

    


    

      
const não impede mutação

      

        const impede trocar a variável inteira.
        Mas o conteúdo interno continua flexível.
        A trava é no rótulo, não na caixa.
      

      

      

    


    

      
Reatribuição x Mutação

      

        Reatribuição: troca o valor todo.

        Mutação: altera apenas partes internas do mesmo objeto.
      

      

      

    


    

      
Cópia x Referência

      

        Primitivos copiam o valor.

        Objetos copiam o endereço (mesmo lugar na memória).
      

      

      

    


    

      
Evitar mutação acidental

      

        Podemos criar cópias com spread (...).

        Também dá para congelar objetos com Object.freeze.
