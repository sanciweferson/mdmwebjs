Primitivos vs Referência em JavaScript

Valores Primitivos

Primitivos são valores simples, como números e textos. Eles não têm “parte interna” para mudar. 

const name = "Sanciweferson"
const age = 29

Aqui name e age existem sozinhos, não dependem de objeto nenhum.

Valores de Referência (Objeto)

Objetos guardam vários valores juntos. Agora name e age vivem dentro de persoa. 

const persoa = {
  name: "san",
  age: 29
}

Quando estão dentro do objeto, precisam do endereço completo: persoa.name e persoa.age.

Mutação em JavaScript

1. Valores primitivos NÃO sofrem mutação

Strings, números, booleanos, null, undefined, symbol e bigint são imutáveis.


    

  

  
  

    
2. Objetos sofrem mutação

    
Objetos e arrays são mutáveis porque guardam referência na memória.

    

    

  

  
  

    
3. const não impede mutação

    
const impede trocar a variável, mas não impede mudar o conteúdo do objeto.
